# 02 - 人机协同（Human-in-the-Loop）

> **核心理念**：在关键步骤暂停，等待人工审核和确认

## 🎯 学习目标

- 理解人机协同的重要性
- 掌握 interrupt 机制
- 学会暂停和恢复 Agent
- 处理用户反馈

---

## 📝 为什么需要人机协同？

### 场景：危险操作需要确认

```typescript
// ❌ 没有人机协同：AI 直接删除数据
用户："删除所有测试数据"
AI: [自动执行] 删除了 1000 条数据  // 危险！

// ✅ 有人机协同：暂停等待确认
用户："删除所有测试数据"
AI: [暂停] 将删除 1000 条数据，是否继续？
用户："确认"
AI: [继续执行] 已删除
```

## 🛠️ Interrupt 基础

### 1. 在节点前暂停

```typescript
import { StateGraph, Annotation } from '@langchain/langgraph';
import { MemorySaver } from '@langchain/langgraph';

const State = Annotation.Root({
  value: Annotation<number>({ default: () => 0 }),
});

function dangerousNode(state: typeof State.State) {
  console.log('⚠️ 执行危险操作...');
  return { value: state.value * 2 };
}

const memory = new MemorySaver();

const graph = new StateGraph(State)
  .addNode('dangerous', dangerousNode)
  .addEdge('__start__', 'dangerous')
  .addEdge('dangerous', '__end__')
  .compile({
    checkpointer: memory,
    interruptBefore: ['dangerous'], // 👈 在 dangerous 前暂停
  });

// 使用
const config = { configurable: { thread_id: 'test' } };

// 第一次调用：暂停在 dangerous 前
const result1 = await graph.invoke({ value: 5 }, config);
console.log('暂停后的状态:', result1); // { value: 5 }

// 用户确认后，继续执行
const result2 = await graph.invoke(null, config); // 👈 传 null 继续
console.log('继续后的状态:', result2); // { value: 10 }
```

---

### 2. 在节点后暂停

```typescript
const graph = new StateGraph(State)
  .addNode('step1', step1Node)
  .addNode('step2', step2Node)
  .addEdge('__start__', 'step1')
  .addEdge('step1', 'step2')
  .addEdge('step2', '__end__')
  .compile({
    checkpointer: memory,
    interruptAfter: ['step1'], // 👈 step1 执行后暂停
  });

// step1 执行完，暂停
// 用户可以查看结果，决定是否继续
```

## 📊 完整示例：删除数据前确认

创建 `src/22-human-in-loop.ts`：

```typescript
import { Annotation, StateGraph } from '@langchain/langgraph';
import { MemorySaver } from '@langchain/langgraph';
import { ChatOpenAI } from '@langchain/openai';
import { HumanMessage, AIMessage, BaseMessage } from '@langchain/core/messages';
import * as readline from 'readline';
import 'dotenv/config';

/**
 * 数据管理 Agent（人机协同）
 * 功能：删除数据前需要人工确认
 */

// State 定义
const DataState = Annotation.Root({
  messages: Annotation<BaseMessage[]>({
    reducer: (x, y) => x.concat(y),
    default: () => [],
  }),
  toDelete: Annotation<number>({
    reducer: (_, y) => y,
    default: () => 0,
  }),
  deleted: Annotation<boolean>({
    reducer: (_, y) => y,
    default: () => false,
  }),
});

type State = typeof DataState.State;

const llm = new ChatOpenAI({ modelName: 'gpt-4o-mini' });

// 节点1：分析删除请求
async function analyzeNode(state: State): Promise<Partial<State>> {
  const userMsg = state.messages[state.messages.length - 1];
  const content = userMsg.content as string;

  // 提取要删除的数量
  const match = content.match(/\d+/);
  const count = match ? parseInt(match[0]) : 0;

  return {
    toDelete: count,
    messages: [new AIMessage(`准备删除 ${count} 条数据，请确认...`)],
  };
}

// 节点2：执行删除（危险操作）
function deleteNode(state: State): Partial<State> {
  console.log(`🗑️ 删除 ${state.toDelete} 条数据...`);

  // 模拟删除
  return {
    deleted: true,
    messages: [new AIMessage(`✅ 已删除 ${state.toDelete} 条数据`)],
  };
}

// 构建图
const memory = new MemorySaver();

const graph = new StateGraph(DataState)
  .addNode('analyze', analyzeNode)
  .addNode('delete', deleteNode)
  .addEdge('__start__', 'analyze')
  .addEdge('analyze', 'delete')
  .addEdge('delete', '__end__')
  .compile({
    checkpointer: memory,
    interruptBefore: ['delete'], // 👈 在删除前暂停
  });

// 测试
async function testHumanInLoop() {
  const threadId = 'delete-task-123';
  const config = { configurable: { thread_id: threadId } };

  console.log('='.repeat(60));
  console.log('👤 用户请求');
  console.log('='.repeat(60));

  // 第一步：发起删除请求
  const step1 = await graph.invoke({ messages: [new HumanMessage('删除 100 条测试数据')] }, config);

  console.log('⏸️  Agent 已暂停，等待确认...');
  console.log('📊 当前状态:', {
    toDelete: step1.toDelete,
    deleted: step1.deleted,
  });

  const lastMsg = step1.messages[step1.messages.length - 1];
  console.log('🤖:', lastMsg.content);

  // 模拟用户确认
  console.log('\n='.repeat(60));
  console.log('👤 用户确认：继续');
  console.log('='.repeat(60));

  // 第二步：继续执行
  const step2 = await graph.invoke(null, config); // 👈 传 null 继续

  console.log('✅ 操作完成');
  console.log('📊 最终状态:', {
    toDelete: step2.toDelete,
    deleted: step2.deleted,
  });

  const finalMsg = step2.messages[step2.messages.length - 1];
  console.log('🤖:', finalMsg.content);
}

testHumanInLoop();
```

```

============================================================
👤 用户确认：继续
============================================================
🗑️ 删除 100 条数据...
✅ 操作完成
📊 最终状态: { toDelete: 100, deleted: true }
🤖: ✅ 已删除 100 条数据
```

## 🎯 实战：交互式命令行工具

创建 `src/23-interactive-cli.ts`：

```typescript
import { Annotation, StateGraph } from '@langchain/langgraph';
import { MemorySaver } from '@langchain/langgraph';
import { HumanMessage, AIMessage, BaseMessage } from '@langchain/core/messages';
import * as readline from 'readline';

/**
 * 交互式 CLI Agent
 * 用户可以随时审查和修改 Agent 的决策
 */

const State = Annotation.Root({
  messages: Annotation<BaseMessage[]>({
    reducer: (x, y) => x.concat(y),
    default: () => [],
  }),
  plan: Annotation<string[]>({
    reducer: (_, y) => y,
    default: () => [],
  }),
});

type StateType = typeof State.State;

// 节点1：制定计划
function planNode(state: StateType): Partial<StateType> {
  const userMsg = state.messages[state.messages.length - 1];
  const task = userMsg.content as string;

  // 简单的任务分解
  const plan = [
    `步骤1: 分析 "${task}"`,
    `步骤2: 查找相关资源`,
    `步骤3: 执行操作`,
    `步骤4: 验证结果`,
  ];

  return {
    plan,
    messages: [new AIMessage(`计划已制定，共 ${plan.length} 步`)],
  };
}

// 节点2：执行计划
function executeNode(state: StateType): Partial<StateType> {
  console.log('\n🚀 执行计划:');
  state.plan.forEach((step, i) => {
    console.log(`  ${i + 1}. ${step}`);
  });

  return {
    messages: [new AIMessage('计划已执行')],
  };
}

// 构建图
const memory = new MemorySaver();

const graph = new StateGraph(State)
  .addNode('plan', planNode)
  .addNode('execute', executeNode)
  .addEdge('__start__', 'plan')
  .addEdge('plan', 'execute')
  .addEdge('execute', '__end__')
  .compile({
    checkpointer: memory,
    interruptAfter: ['plan'], // 计划制定后暂停
  });

// 命令行交互
const rl = readline.createInterface({
  input: process.stdin,
  output: process.stdout,
});

function question(prompt: string): Promise<string> {
  return new Promise(resolve => {
    rl.question(prompt, resolve);
  });
}

async function interactiveCLI() {
  const threadId = 'cli-session';
  const config = { configurable: { thread_id: threadId } };

  console.log('🤖 交互式 Agent 启动\n');

  // 获取用户任务
  const task = await question('👤 请输入任务: ');

  // 第一步：制定计划
  const step1 = await graph.invoke({ messages: [new HumanMessage(task)] }, config);

  console.log('\n📋 Agent 制定的计划:');
  step1.plan.forEach((p: string, i: number) => {
    console.log(`  ${i + 1}. ${p}`);
  });

  // 询问用户是否继续
  const confirm = await question('\n👤 是否继续执行？(y/n): ');

  if (confirm.toLowerCase() === 'y') {
    // 第二步：执行计划
    const step2 = await graph.invoke(null, config);

    console.log('\n✅ 任务完成');
  } else {
    console.log('\n❌ 任务已取消');
  }

  rl.close();
}

interactiveCLI();
```

## 🔍 高级技巧

### 1. 修改 State 后继续

```typescript
// 用户可以修改 State 再继续
const state = await graph.getState(config);

// 修改计划
state.values.plan = ['新步骤1', '新步骤2'];

// 更新 State
await graph.updateState(config, state.values);

// 继续执行
await graph.invoke(null, config);
```

### 2. 回退到之前的步骤

```typescript
// 获取历史 checkpoints
const history = await graph.getStateHistory(config);

// 回退到第 2 个 checkpoint
const secondCheckpoint = history[1];
await graph.invoke(null, {
  configurable: {
    thread_id: threadId,
    checkpoint_id: secondCheckpoint.config.configurable.checkpoint_id,
  },
});
```

## 💡 常见问题 FAQ

**Q: 可以在多个节点前暂停吗？**
A: 可以！`interruptBefore: ['node1', 'node2', 'node3']`

**Q: 如何取消执行？**
A: 不调用 `invoke(null, config)` 即可。或者删除 checkpoint。

**Q: 可以暂停后修改用户输入吗？**
A: 可以！使用 `updateState` 修改 messages。
