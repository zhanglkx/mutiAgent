## 📝 什么是状态机？

### 生活中的状态机

**红绿灯**：

```
[红灯] → 30秒 → [绿灯] → 20秒 → [黄灯] → 3秒 → [红灯]
  ↑_______________________________________________|
```

- **状态**：红灯、绿灯、黄灯
- **转换规则**：时间触发
- **循环**：不断重复

**订单状态**：

```
[待支付] → 支付成功 → [待发货] → 发货 → [待收货] → 确认收货 → [已完成]
```

### 程序中的状态机

**前端类比：Redux**

```typescript
// Redux Store（全局状态）
interface AppState {
  user: User | null;
  cart: CartItem[];
  currentPage: string;
}

// Reducer（状态更新规则）
function reducer(state: AppState, action: Action): AppState {
  switch (action.type) {
    case 'ADD_TO_CART':
      return { ...state, cart: [...state.cart, action.item] };
    case 'NAVIGATE':
      return { ...state, currentPage: action.page };
  }
}
```

**LangGraph State（Agent 的状态）**

```typescript
import { BaseMessage } from '@langchain/core/messages';

// Agent State（Agent 的记忆）
interface AgentState {
  messages: BaseMessage[];     // 对话历史
  currentStep: string;          // 当前步骤
  toolResults: Record<string, any>;  // 工具执行结果
}

// Node（状态更新规则）
function thinkNode(state: AgentState): Partial<AgentState> {
  return { currentStep: 'use_tool' };
}
```

## 🛠️ LangGraph 中的 State

### 1. 基本 State 定义

```typescript
import { Annotation } from '@langchain/langgraph';
import { BaseMessage } from '@langchain/core/messages';

// 定义 State 结构
const StateAnnotation = Annotation.Root({
  messages: Annotation<BaseMessage[]>({
    reducer: (current, update) => current.concat(update),  // 追加消息
  }),
});

type AgentState = typeof StateAnnotation.State;
```

**解释**：
- `messages`：对话历史
- `reducer`：如何合并新旧状态（追加而非覆盖）

---

### 2. Reducer 的重要性

**问题**：如果没有 reducer，会发生什么？

```typescript
// ❌ 没有 reducer（默认覆盖）
const StateAnnotation = Annotation.Root({
  messages: Annotation<BaseMessage[]>(),  // 默认：覆盖
});

// 节点 A 返回：{ messages: [msg1] }
// 节点 B 返回：{ messages: [msg2] }
// 最终结果：{ messages: [msg2] }  // msg1 丢失了！
```

```typescript
// ✅ 有 reducer（追加）
const StateAnnotation = Annotation.Root({
  messages: Annotation<BaseMessage[]>({
    reducer: (current, update) => current.concat(update),
  }),
});

// 节点 A 返回：{ messages: [msg1] }
// 节点 B 返回：{ messages: [msg2] }
// 最终结果：{ messages: [msg1, msg2] }  // ✅ 都保留了
```

**前端类比**：

```typescript
// React 的 setState
const [items, setItems] = useState([1, 2, 3]);

// ❌ 覆盖（错误）
setItems([4]);  // 结果：[4]，前面的丢失了

// ✅ 追加（正确）
setItems([...items, 4]);  // 结果：[1, 2, 3, 4]
```

---

### 3. 完整的 State 示例

```typescript
import { Annotation } from '@langchain/langgraph';
import { BaseMessage } from '@langchain/core/messages';

const AgentState = Annotation.Root({
  // 1. 对话历史（追加）
  messages: Annotation<BaseMessage[]>({
    reducer: (current, update) => current.concat(update),
  }),

  // 2. 当前步骤（覆盖）
  currentStep: Annotation<string>({
    reducer: (_, update) => update,  // 直接覆盖
  }),

  // 3. 工具结果（合并）
  toolResults: Annotation<Record<string, any>>({
    reducer: (current, update) => ({ ...current, ...update }),
  }),

  // 4. 循环计数（累加）
  loopCount: Annotation<number>({
    reducer: (current, update) => current + update,
    default: () => 0,  // 默认值
  }),
});

type State = typeof AgentState.State;
```

## 📊 完整示例：天气查询 Agent

创建 `src/12-weather-agent-state.ts`：

```typescript
import { Annotation, StateGraph } from '@langchain/langgraph';
import { BaseMessage, HumanMessage, AIMessage } from '@langchain/core/messages';
import { ChatOpenAI } from '@langchain/openai';
import 'dotenv/config';

/**
 * 天气查询 Agent
 * 演示 State 的定义和使用
 */

// 1. 定义 State
const WeatherState = Annotation.Root({
  messages: Annotation<BaseMessage[]>({
    reducer: (x, y) => x.concat(y),
  }),

  city: Annotation<string | null>({
    reducer: (_, y) => y,
    default: () => null,
  }),

  weather: Annotation<string | null>({
    reducer: (_, y) => y,
    default: () => null,
  }),

  step: Annotation<string>({
    reducer: (_, y) => y,
    default: () => 'extract_city',
  }),
});

type State = typeof WeatherState.State;

// 2. 定义节点
const llm = new ChatOpenAI({ modelName: 'gpt-4o-mini' });

// 节点1：提取城市名
async function extractCityNode(state: State): Promise<Partial<State>> {
  const userMessage = state.messages[state.messages.length - 1];

  const response = await llm.invoke([
    new HumanMessage(
      `从以下消息中提取城市名（只返回城市名，不要其他内容）：\n${userMessage.content}`
    ),
  ]);

  const city = (response.content as string).trim();

  return {
    city,
    step: 'fetch_weather',
    messages: [new AIMessage(`识别到城市：${city}`)],
  };
}

// 节点2：查询天气
async function fetchWeatherNode(state: State): Promise<Partial<State>> {
  // 模拟天气 API
  const weatherData: Record<string, string> = {
    '北京': '25°C，晴天',
    '上海': '28°C，多云',
    '广州': '32°C，雷阵雨',
  };

  const weather = weatherData[state.city!] || '暂无数据';

  return {
    weather,
    step: 'respond',
  };
}

// 节点3：生成回复
async function respondNode(state: State): Promise<Partial<State>> {
  const message = `${state.city}的天气是：${state.weather}`;

  return {
    messages: [new AIMessage(message)],
    step: 'end',
  };
}

// 3. 构建图
const graph = new StateGraph(WeatherState)
  .addNode('extract_city', extractCityNode)
  .addNode('fetch_weather', fetchWeatherNode)
  .addNode('respond', respondNode)
  .addEdge('__start__', 'extract_city')
  .addEdge('extract_city', 'fetch_weather')
  .addEdge('fetch_weather', 'respond')
  .addEdge('respond', '__end__')
  .compile();

// 4. 测试
async function main() {
  const result = await graph.invoke({
    messages: [new HumanMessage('北京今天天气怎么样？')],
  });

  console.log('='.repeat(50));
  console.log('最终状态：');
  console.log('='.repeat(50));
  console.log('城市：', result.city);
  console.log('天气：', result.weather);
  console.log('对话历史：');
  result.messages.forEach((msg, i) => {
    console.log(`  ${i + 1}. [${msg._getType()}]: ${msg.content}`);
  });
}

main();
```

**运行结果**：

```
==================================================
最终状态：
==================================================
城市： 北京
天气： 25°C，晴天
对话历史：
  1. [human]: 北京今天天气怎么样？
  2. [ai]: 识别到城市：北京
  3. [ai]: 北京的天气是：25°C，晴天
```

## 🔍 State 设计最佳实践

### 1. 只存必要的数据

```typescript
// ❌ 不好：存储太多冗余数据
const BadState = Annotation.Root({
  messages: Annotation<BaseMessage[]>(),
  lastMessage: Annotation<string>(),         // 冗余！
  messageCount: Annotation<number>(),        // 冗余！
  firstMessage: Annotation<string>(),        // 冗余！
});

// ✅ 好：只存核心数据
const GoodState = Annotation.Root({
  messages: Annotation<BaseMessage[]>(),     // 其他都可以计算出来
});

// 需要时再计算
const lastMessage = state.messages[state.messages.length - 1];
const messageCount = state.messages.length;
```

### 2. 合理使用 default

```typescript
const State = Annotation.Root({
  // ✅ 数组用空数组
  messages: Annotation<BaseMessage[]>({
    default: () => [],
  }),

  // ✅ 对象用空对象
  toolResults: Annotation<Record<string, any>>({
    default: () => ({}),
  }),

  // ✅ 数字用 0
  loopCount: Annotation<number>({
    default: () => 0,
  }),

  // ✅ 可选值用 null
  currentTask: Annotation<string | null>({
    default: () => null,
  }),
});
```

### 3. 选择合适的 reducer

```typescript
const State = Annotation.Root({
  // 追加（列表、数组）
  messages: Annotation<BaseMessage[]>({
    reducer: (x, y) => x.concat(y),
  }),

  // 覆盖（单值）
  currentStep: Annotation<string>({
    reducer: (_, y) => y,
  }),

  // 合并（对象）
  metadata: Annotation<Record<string, any>>({
    reducer: (x, y) => ({ ...x, ...y }),
  }),

  // 累加（计数器）
  count: Annotation<number>({
    reducer: (x, y) => x + y,
  }),

  // 取最大值
  maxScore: Annotation<number>({
    reducer: (x, y) => Math.max(x, y),
  }),
});
```

## 📚 知识点总结

### ✅ 你已经掌握

1. **状态机概念**
   - State：数据存储
   - Reducer：更新规则
   - 节点：处理逻辑

2. **State 定义**
   - 使用 `Annotation.Root`
   - 设置 reducer
   - 设置 default

3. **Reducer 类型**
   - 追加：`concat`
   - 覆盖：`(_, y) => y`
   - 合并：`{ ...x, ...y }`
   - 累加：`x + y`

4. **设计原则**
   - 只存必要数据
   - 合理使用 default
   - 选择合适的 reducer

### 🔄 前端类比汇总

| LangGraph 概念 | 前端类比 | 说明 |
|---------------|---------|------|
| State | Redux Store | 全局状态 |
| Annotation.Root | interface | 类型定义 |
| reducer | Reducer | 更新逻辑 |
| default | 初始值 | 默认数据 |
| 节点返回值 | dispatch(action) | 触发更新 |

## 💡 常见问题 FAQ

**Q: 为什么要用 Annotation 而不是普通 interface？**
A: Annotation 包含了 reducer 逻辑，能自动处理状态合并。普通 interface 只有类型信息。

**Q: 可以在运行时修改 State 结构吗？**
A: 不可以。State 结构在图编译时就固定了，运行时只能修改值。

**Q: 如果 reducer 太复杂怎么办？**
A: 把复杂逻辑放到节点函数中，reducer 只做简单的合并。

**Q: State 可以嵌套吗？**
A: 可以，但不推荐过度嵌套。保持扁平化更易维护。

```typescript
// ✅ 扁平化
const State = Annotation.Root({
  userName: Annotation<string>(),
  userAge: Annotation<number>(),
});

// ❌ 过度嵌套
const State = Annotation.Root({
  user: Annotation<{
    name: string;
    profile: {
      age: number;
      address: {
        city: string;
      };
    };
  }>(),
});
```
