## 📝 四种核心消息类型

LangChain 的消息系统类似于前端的组件系统，每种消息类型有特定职责：

### 1. SystemMessage - 系统指令（全局配置）

**前端类比**：类似 React 的 `Context Provider`，设置全局配置

```typescript
import { SystemMessage } from '@langchain/core/messages';

const systemMsg = new SystemMessage('你是一个专业的前端工程师助手');
```

**作用**：
- 定义 AI 的角色和行为
- 设置输出格式规则
- 永远在对话最前面

**最佳实践**：

```typescript
// ✅ 好的 System Message
const goodSystem = new SystemMessage(`你是一个 React 专家。
规则：
1. 代码示例使用 TypeScript
2. 优先使用函数组件和 Hooks
3. 回答要简洁，不超过 200 字
4. 代码要有注释`);

// ❌ 不好的 System Message
const badSystem = new SystemMessage('你是 AI');  // 太模糊
```

---

### 2. HumanMessage - 用户消息（输入）

**前端类比**：用户在表单里输入的内容

```typescript
import { HumanMessage } from '@langchain/core/messages';

const userMsg = new HumanMessage('如何使用 useState？');
```

**特点**：
- 代表用户的问题或指令
- 可以包含文字、图片（多模态）
- 每轮对话至少有一个

**多模态示例**（GPT-4V / Claude 3.5 支持）：

```typescript
// 图片 + 文字
const multimodalMsg = new HumanMessage({
  content: [
    {
      type: 'text',
      text: '这张图片里有什么？',
    },
    {
      type: 'image_url',
      image_url: {
        url: 'https://example.com/image.jpg',
      },
    },
  ],
});

await llm.invoke([multimodalMsg]);
```

---

### 3. AIMessage - AI 回复（输出）

**前端类比**：后端 API 返回的数据

```typescript
import { AIMessage } from '@langchain/core/messages';

const aiMsg = new AIMessage('useState 是 React 的状态管理 Hook...');
```

**特点**：
- 模型的响应内容
- 自动由 `llm.invoke()` 返回
- 可以包含工具调用信息（第五课详解）

**手动构造场景**（模拟历史对话）：

```typescript
// 假设要恢复一个历史对话
const conversation = [
  new HumanMessage('你好'),
  new AIMessage('你好！有什么可以帮你？'),
  new HumanMessage('介绍一下 React'),
];

const response = await llm.invoke(conversation);
```

---

### 4. ToolMessage - 工具执行结果（函数返回值）

**前端类比**：API 调用的返回结果

```typescript
import { ToolMessage } from '@langchain/core/messages';

const toolMsg = new ToolMessage({
  content: '当前天气：25°C，晴天',
  tool_call_id: 'call_123',  // 关联到哪次工具调用
});
```

**使用场景**：
- AI 调用函数后，将结果传回给 AI
- 构建 ReAct 循环的关键（第二阶段重点）

**完整流程示例**：

```typescript
// 1. 用户问天气
new HumanMessage('北京天气怎么样？');

// 2. AI 决定调用工具（自动生成）
new AIMessage({
  content: '',
  tool_calls: [{
    name: 'get_weather',
    args: { city: '北京' },
    id: 'call_123',
  }],
});

// 3. 你执行函数，返回结果
new ToolMessage({
  content: '25°C，晴天',
  tool_call_id: 'call_123',
});

// 4. AI 总结答案
new AIMessage('北京今天 25°C，晴天');
```

### 示例 1：客服机器人

创建 `src/03-customer-service.ts`：

```typescript
import { ChatOpenAI } from '@langchain/openai';
import {
  SystemMessage,
  HumanMessage,
  AIMessage,
  BaseMessage,
} from '@langchain/core/messages';
import 'dotenv/config';

/**
 * 客服机器人示例
 * 演示如何用不同消息类型构建专业对话
 */

const llm = new ChatOpenAI({ modelName: 'gpt-4o-mini' });

// 系统设定：定义角色和规则
const systemPrompt = new SystemMessage(`你是某电商平台的客服机器人。
规则：
1. 礼貌专业，每次回复包含问候语
2. 如果涉及退款，提醒需要提供订单号
3. 回复不超过 100 字
4. 使用 emoji 增加亲和力`);

// 模拟多轮对话
async function main() {
  const conversation: BaseMessage[] = [systemPrompt];

  // 第一轮
  conversation.push(new HumanMessage('我的包裹怎么还没到？'));
  const response1 = await llm.invoke(conversation);
  console.log(`客服: ${response1.content}\n`);
  conversation.push(new AIMessage(response1.content as string));

  // 第二轮
  conversation.push(new HumanMessage('订单号是 123456'));
  const response2 = await llm.invoke(conversation);
  console.log(`客服: ${response2.content}\n`);
  conversation.push(new AIMessage(response2.content as string));

  // 第三轮
  conversation.push(new HumanMessage('那我要退款'));
  const response3 = await llm.invoke(conversation);
  console.log(`客服: ${response3.content}\n`);
}

main();
```


### 示例 2：消息历史管理

**问题**：对话历史太长，Token 消耗大，成本高！

**解决方案**：保留最近 N 轮对话

创建 `src/04-conversation-manager.ts`：

```typescript
import { ChatOpenAI } from '@langchain/openai';
import {
  SystemMessage,
  HumanMessage,
  AIMessage,
  BaseMessage,
} from '@langchain/core/messages';
import 'dotenv/config';

/**
 * 滑动窗口管理对话历史
 * 前端类比：类似虚拟列表，只渲染可见部分
 */

class ConversationManager {
  private systemMessage: SystemMessage | null = null;
  private history: BaseMessage[] = [];
  private maxTurns: number;

  constructor(maxTurns = 5) {
    this.maxTurns = maxTurns;
  }

  setSystemMessage(content: string) {
    this.systemMessage = new SystemMessage(content);
  }

  addUserMessage(content: string) {
    this.history.push(new HumanMessage(content));
    this.trimHistory();
  }

  addAIMessage(content: string) {
    this.history.push(new AIMessage(content));
    this.trimHistory();
  }

  /**
   * 保持历史长度在限制内
   * 类似 React 的 useMemo 优化
   */
  private trimHistory() {
    const maxMessages = this.maxTurns * 2; // 一轮 = 1 Human + 1 AI
    if (this.history.length > maxMessages) {
      this.history = this.history.slice(-maxMessages);
    }
  }

  /**
   * 获取完整消息列表（用于发送给 LLM）
   */
  getMessages(): BaseMessage[] {
    const messages: BaseMessage[] = [];
    if (this.systemMessage) {
      messages.push(this.systemMessage);
    }
    messages.push(...this.history);
    return messages;
  }

  clear() {
    this.history = [];
  }

  getHistoryLength() {
    return this.history.length;
  }
}

// 使用示例
async function main() {
  const llm = new ChatOpenAI({ modelName: 'gpt-4o-mini' });
  const manager = new ConversationManager(3); // 只保留 3 轮对话

  manager.setSystemMessage('你是一个简洁的助手');

  for (let i = 0; i < 10; i++) {
    manager.addUserMessage(`这是第 ${i + 1} 条消息`);
    const response = await llm.invoke(manager.getMessages());
    manager.addAIMessage(response.content as string);

    console.log(`轮次 ${i + 1}：历史消息数量 = ${manager.getHistoryLength()}`);
  }

  // 只会保留最后 3 轮（6 条消息）
  console.log(`\n最终历史消息数量: ${manager.getHistoryLength()}`); // 输出：6
}

main();
```

**前端类比**：

```typescript
// React 版本
const [messages, setMessages] = useState<Message[]>([]);
const MAX_HISTORY = 10;

const addMessage = (msg: Message) => {
  setMessages((prev) => {
    const newMessages = [...prev, msg];
    // 只保留最新 10 条
    return newMessages.slice(-MAX_HISTORY);
  });
};
```

### 技巧 1：动态 System Message

根据用户类型调整 AI 行为：

```typescript
type UserLevel = 'beginner' | 'intermediate' | 'expert';

function createSystemMessage(level: UserLevel): SystemMessage {
  const prompts: Record<UserLevel, string> = {
    beginner: '你是一个耐心的老师，用简单的语言解释技术概念，多举例子。',
    intermediate: '你是一个技术导师，提供最佳实践和进阶建议。',
    expert: '你是技术同行，直接给出技术细节和性能优化建议。',
  };

  return new SystemMessage(prompts[level]);
}

// 使用
const messages = [
  createSystemMessage('expert'),
  new HumanMessage('解释 React Fiber 架构'),
];
```

### 技巧 2：消息压缩

长对话可以总结压缩：

```typescript
async function summarizeConversation(
  messages: BaseMessage[]
): Promise<SystemMessage> {
  const llm = new ChatOpenAI({ modelName: 'gpt-4o-mini' });

  const summaryPrompt = `请总结以下对话的关键信息：

${messages.map((m) => `${m._getType()}: ${m.content}`).join('\n')}

总结要点（不超过 50 字）：`;

  const summary = await llm.invoke(summaryPrompt);
  return new SystemMessage(`前情提要：${summary.content}`);
}

// 每 20 轮对话压缩一次
async function compressIfNeeded(conversation: BaseMessage[]) {
  if (conversation.length > 40) {
    const summary = await summarizeConversation(conversation.slice(1));
    return [conversation[0], summary, ...conversation.slice(-10)];
  }
  return conversation;
}
```

### 技巧 3：消息模板类

创建可复用的消息模板：

```typescript
/**
 * 消息模板库
 * 类似 React 的自定义 Hooks
 */
class MessageTemplates {
  /**
   * 代码审查模式
   */
  static codeReview(): SystemMessage {
    return new SystemMessage(`你是代码审查专家。
审查标准：
1. 代码规范
2. 性能优化
3. 安全漏洞
4. 可维护性

输出格式：
✅ 优点：...
⚠️ 问题：...
💡 建议：...`);
  }

  /**
   * 代码讲解模式
   */
  static explainCode(): SystemMessage {
    return new SystemMessage(`你是代码讲解专家。
讲解风格：
1. 先概述整体逻辑
2. 逐行解释关键代码
3. 用类比帮助理解
4. 标注最佳实践`);
  }

  /**
   * 调试助手模式
   */
  static debugHelper(): SystemMessage {
    return new SystemMessage(`你是调试专家。
分析步骤：
1. 定位错误行
2. 分析错误原因
3. 提供修复方案
4. 给出预防建议`);
  }
}

// 使用

];
```

## 📊 完整项目：智能编程助手

创建 `src/05-coding-assistant.ts`：

```typescript
import { ChatOpenAI } from '@langchain/openai';
import { SystemMessage, HumanMessage } from '@langchain/core/messages';
import 'dotenv/config';

/**
 * 智能编程助手
 * 功能：代码审查、解释、调试
 * 演示消息系统的综合应用
 */

type Mode = 'review' | 'explain' | 'debug';

class CodingAssistant {
  private llm: ChatOpenAI;
  private modes: Record<Mode, string>;

  constructor() {
    this.llm = new ChatOpenAI({
      modelName: 'gpt-4o-mini',
      temperature: 0.3,
    });

    this.modes = {
      review: `你是代码审查专家。
审查重点：
1. 代码规范（命名、格式）
2. 潜在 bug
3. 性能问题
4. 安全漏洞

输出格式：
✅ 优点：
⚠️ 问题：
💡 建议：`,

      explain: `你是代码讲解专家。
讲解步骤：
1. 概述功能（1 句话）
2. 逐行解释
3. 类比说明
4. 最佳实践

使用 emoji 和清晰的排版。`,

      debug: `你是调试专家。
分析流程：
1. 🔍 定位错误
2. 🧠 分析原因
3. 🛠️ 修复代码
4. 📚 预防建议

给出可直接运行的修复代码。`,
    };
  }

  async process(mode: Mode, code: string): Promise<string> {
    const messages = [
      new SystemMessage(this.modes[mode]),
      new HumanMessage(`代码：\n\`\`\`typescript\n${code}\n\`\`\``),
    ];

    const response = await this.llm.invoke(messages);
    return response.content as string;
  }
}

// 使用示例
async function main() {
  const assistant = new CodingAssistant();

  // 示例代码（有 bug）
  const buggyCode = `
async function fetchData() {
  let data = null;
  fetch('/api/users')
    .then(res => res.json())
    .then(json => data = json);

  return data;  // bug: 返回 null
}
`;

  console.log('='.repeat(50));
  console.log('🐛 调试模式');
  console.log('='.repeat(50));
  console.log(await assistant.process('debug', buggyCode));

  console.log('\n' + '='.repeat(50));
  console.log('📖 解释模式');
  console.log('='.repeat(50));

  const goodCode = `
const [count, setCount] = useState(0);

useEffect(() => {
  document.title = \`Count: \${count}\`;
}, [count]);
`;

  console.log(await assistant.process('explain', goodCode));
}

main();
```

## 💡 常见问题 FAQ

**Q: System Message 可以有多个吗？**
A: 技术上可以，但不推荐。多个会混淆模型。正确做法是合并为一个。

**Q: 对话历史多长合适？**
A:
- 简单问答：5-10 轮
- 复杂任务：20-30 轮
- 超过 50 轮建议总结压缩

**Q: 如何处理图片消息？**
A: 需要使用支持多模态的模型（GPT-4V、Claude 3.5）：
```typescript
new HumanMessage({
  content: [
    { type: 'text', text: '分析这张图' },
    { type: 'image_url', image_url: { url: '...' } },
  ],
});
```

**Q: AIMessage 的 tool_calls 是什么？**
A: 这是第五课的重点内容，用于函数调用（Function Calling）。
