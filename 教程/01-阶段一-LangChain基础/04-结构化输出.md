# 04 - 结构化输出

> **核心理念**：让 AI 返回标准 JSON，就像调用类型安全的 API

## 🎯 学习目标

- 理解为什么需要结构化输出
- 掌握 Zod Schema 验证
- 学会使用 `withStructuredOutput`
- 构建类型安全的 AI 应用

---

## 📝 为什么需要结构化输出？

### 问题：AI 返回的是自由文本

```typescript
const response = await llm.invoke('提取文本中的人名和年龄：小明今年25岁');
console.log(response.content);
// 可能返回："人名是小明，年龄是25岁"
// 也可能返回："小明，25"
// 甚至可能返回："根据文本，我们可以知道..."
```

**问题**：

- ❌ 格式不统一，难以解析
- ❌ 需要写正则表达式提取数据
- ❌ 容易出错

---

### 解决方案：强制 AI 返回 JSON

```typescript
import { z } from 'zod';
import { ChatOpenAI } from '@langchain/openai';

// 1. 定义数据结构（使用 Zod）
const personSchema = z.object({
  name: z.string().describe('人名'),
  age: z.number().describe('年龄'),
});

// 2. 让 LLM 支持结构化输出
const llm = new ChatOpenAI({
  modelName: 'gpt-4o-mini',
}).withStructuredOutput(personSchema);

// 3. 调用（返回类型安全的对象！）
const result = await llm.invoke('小明今年25岁');
console.log(result);
// ✅ 输出：{ name: "小明", age: 25 }

// TypeScript 自动推断类型！
console.log(result.name.toUpperCase()); // ✅ 类型安全
```

**前端类比**：

```typescript
// 调用普通 API（不确定返回什么）
const response = await fetch('/api/user');
const data = await response.json(); // any 类型

// 调用类型安全的 API
interface User {
  name: string;
  age: number;
}
const data: User = await fetchUser(); // 类型安全！
```

## 🛠️ Zod Schema 基础

### 1. 基本类型

```typescript
import { z } from 'zod';

// 字符串
const nameSchema = z.string();

// 数字
const ageSchema = z.number();

// 布尔值
const isActiveSchema = z.boolean();

// 日期
const createdAtSchema = z.date();

// 枚举
const roleSchema = z.enum(['admin', 'user', 'guest']);
```

### 2. 对象和数组

```typescript
// 对象
const userSchema = z.object({
  name: z.string(),
  age: z.number(),
  email: z.string().email(),
});

// 数组
const usersSchema = z.array(userSchema);

// 嵌套对象
const companySchema = z.object({
  name: z.string(),
  founder: z.object({
    name: z.string(),
    age: z.number(),
  }),
});
```

### 3. 可选和默认值

```typescript
const userSchema = z.object({
  name: z.string(),
  age: z.number().optional(), // 可选字段
  role: z.string().default('user'), // 默认值
  bio: z.string().nullable(), // 可以为 null
});
```

### 4. 添加描述（提示 AI）

```typescript
const productSchema = z.object({
  name: z.string().describe('产品名称'),
  price: z.number().describe('价格（单位：元）'),
  category: z.enum(['electronics', 'books', 'clothing']).describe('产品类别'),
});

// AI 会根据描述理解每个字段的含义
```

## 🎨 实战示例

### 示例 1：信息提取

创建 `src/08-extract-info.ts`：

```typescript
import { ChatOpenAI } from '@langchain/openai';
import { z } from 'zod';
import 'dotenv/config';

/**
 * 从文本中提取结构化信息
 * 前端类比：表单验证
 */

// 定义 Schema
const contactSchema = z.object({
  name: z.string().describe('联系人姓名'),
  email: z.string().email().describe('邮箱地址'),
  phone: z.string().optional().describe('电话号码'),
  company: z.string().optional().describe('公司名称'),
});

// 创建支持结构化输出的 LLM
const llm = new ChatOpenAI({
  modelName: 'gpt-4o-mini',
}).withStructuredOutput(contactSchema);

// 测试
async function extractContact(text: string) {
  const result = await llm.invoke(`从以下文本提取联系人信息：\n\n${text}`);
  return result;
}

// 使用
const text = `
你好，我是张三，在字节跳动工作。
我的邮箱是 zhangsan@example.com，
手机号是 13800138000。
`;

const contact = await extractContact(text);
console.log(contact);
// 输出：
// {
//   name: "张三",
//   email: "zhangsan@example.com",
//   phone: "13800138000",
//   company: "字节跳动"
// }

// TypeScript 类型安全！
console.log(contact.email.toLowerCase()); // ✅
```

## 📊 完整项目：简历解析器

创建 `src/09-resume-parser.ts`：

```typescript
import { ChatOpenAI } from '@langchain/openai';
import { z } from 'zod';
import 'dotenv/config';

/**
 * 简历解析器
 * 功能：从简历文本中提取结构化信息
 */

// 定义简历结构
const resumeSchema = z.object({
  personal: z.object({
    name: z.string(),
    email: z.string().email(),
    phone: z.string().optional(),
    location: z.string().optional(),
  }),

  education: z.array(
    z.object({
      school: z.string(),
      degree: z.string(),
      major: z.string(),
      startYear: z.number(),
      endYear: z.number(),
    })
  ),

  experience: z.array(
    z.object({
      company: z.string(),
      position: z.string(),
      startDate: z.string(),
      endDate: z.string().optional(),
      description: z.string(),
    })
  ),

  skills: z.array(z.string()),

  languages: z.array(
    z.object({
      name: z.string(),
      level: z.enum(['native', 'fluent', 'intermediate', 'basic']),
    })
  ),
});

// 创建解析器
class ResumeParser {
  private llm: ChatOpenAI;

  constructor() {
    this.llm = new ChatOpenAI({
      modelName: 'gpt-4o-mini',
      temperature: 0, // 提取数据要准确
    }).withStructuredOutput(resumeSchema);
  }

  async parse(resumeText: string) {
    return await this.llm.invoke(`
请从以下简历中提取结构化信息：

${resumeText}
    `);
  }
}

// 测试
const resumeText = `
张三
邮箱：zhangsan@example.com
电话：13800138000
地点：北京

教育经历：
- 清华大学，计算机科学与技术，本科，2016-2020
- 斯坦福大学，人工智能，硕士，2020-2022

工作经历：
- 字节跳动，高级前端工程师，2022.06-至今
  负责抖音前端架构设计和开发，使用 React、TypeScript、Webpack

技能：
JavaScript, TypeScript, React, Vue, Node.js, Python

语言：
- 中文（母语）
- 英语（流利）
`;

async function main() {
  const parser = new ResumeParser();
  const resume = await parser.parse(resumeText);

  console.log('解析结果：');
  console.log(JSON.stringify(resume, null, 2));

  // 类型安全的访问
  console.log(`\n姓名：${resume.personal.name}`);
  console.log(`最高学历：${resume.education[resume.education.length - 1].degree}`);
  console.log(`当前公司：${resume.experience[0].company}`);
  console.log(`技能数量：${resume.skills.length}`);
}

main();
```

## 🔍 错误处理

### 1. 验证失败处理

```typescript
import { ChatOpenAI } from '@langchain/openai';
import { z } from 'zod';

const strictSchema = z.object({
  age: z.number().min(0).max(150),
  email: z.string().email(),
});

const llm = new ChatOpenAI({
  modelName: 'gpt-4o-mini',
}).withStructuredOutput(strictSchema);

try {
  // 如果 AI 返回的数据不符合 Schema，会抛出错误
  const result = await llm.invoke('年龄是 999，邮箱是 invalid');
  console.log(result);
} catch (error) {
  console.error('验证失败：', error);
  // 可以让 AI 重新生成
}
```

### 2. 提供示例（提高准确率）

```typescript
const llm = new ChatOpenAI({
  modelName: 'gpt-4o-mini',

// 在 prompt 中提供示例
const result = await llm.invoke(`
从文本提取用户信息。

示例格式：
{
  "name": "张三",
  "age": 25,
  "email": "zhangsan@example.com"
}

文本：李四，30岁，邮箱 lisi@example.com
`);
```

## 💡 常见问题 FAQ

**Q: 所有模型都支持结构化输出吗？**
A: 不是。需要支持 Function Calling 的模型：

- ✅ OpenAI: gpt-4, gpt-4o, gpt-4o-mini
- ✅ Claude: claude-3-5-sonnet, claude-3-opus
- ❌ 部分老模型不支持

**Q: 结构化输出会增加成本吗？**
A: 会稍微增加，因为需要在 system message 中添加 Schema 定义。但换来的是更高的准确性和更少的解析代码。

**Q: 如果 AI 返回的数据不完整怎么办？**
A:

1. 使用 `.optional()` 标记可选字段
2. 在 prompt 中强调必填字段
3. 设置 `temperature=0` 提高一致性

**Q: Zod Schema 可以复用吗？**
A: 当然！建议把常用的 Schema 定义在单独的文件中：

```typescript
// schemas/user.schema.ts
export const userSchema = z.object({
  name: z.string(),
  email: z.string().email(),
});

// 在其他地方引用
import { userSchema } from './schemas/user.schema';
```

**Q: 如何处理复杂的嵌套结构？**
A: 分步构建：

```typescript
const addressSchema = z.object({
  street: z.string(),
  city: z.string(),
});

const userSchema = z.object({
  name: z.string(),
  address: addressSchema, // 引用其他 Schema
});
```
